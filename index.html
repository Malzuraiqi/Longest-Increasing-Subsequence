What is LIS?

The Longest Increasing Subsequence (LIS) problem asks: given a sequence of numbers, find the longest subsequence where each element is strictly greater than the one before it.

In [3, 1, 4, 1, 5, 9, 2, 6], one LIS is [1, 4, 5, 9] with length 4. Elements don't have to be adjacent — they just need to preserve order and be increasing.

Why Does It Matter?

LIS is a classic showcase for Dynamic Programming and the power of binary search optimization. It appears in real-world problems:

• Bioinformatics: DNA/RNA alignment
• Scheduling: box stacking problems
• Version control: diff/patch algorithms
• Finance: stock trading strategies

Two Approaches

DP O(n²) — For each element i, scan all j < i. If A[j] < A[i], try extending the LIS at j. Simple and illustrates optimal substructure perfectly.

Patience Sort O(n log n) — Use binary search to place each element on a pile. The number of piles equals the LIS length. Binary search cuts inner loop from O(n) to O(log n).

Subsequence vs Substring: A subsequence deletes elements but preserves order. [1,4,9] is a subsequence of [3,1,4,1,5,9]. A substring must be contiguous — adjacent elements only.
Optimal Substructure: The longest IS ending at index i depends only on the longest IS ending at indices 0..i-1. We can build big solutions from stored smaller ones — the hallmark of Dynamic Programming.
Overlapping Subproblems: dp[j] is consulted by every future dp[i] where A[j] < A[i]. Without DP, we'd recompute these subproblems exponentially. Memoization turns O(2ⁿ) into O(n²).
Binary Search Key Insight: In patience sort, pile tops are always in sorted order. This invariant lets us binary-search for the correct pile in O(log n), transforming the O(n²) DP into O(n log n).

Why does binary search save us? The DP inner loop scans O(n) elements. Patience sort replaces this with O(log n) binary search on the pile tops — which are always sorted. Total: n × log n.

CLO-2: Role of Binary Search

Key invariant: Pile tops are always in strictly increasing order. This is maintained because we only replace a pile top with a smaller or equal value, and new piles are always larger than all existing tops.

Since pile tops are sorted, we can use binary search to find the leftmost pile whose top ≥ current element. We search for the position where piles[mid] ≥ val.

CLO-3: Dynamic Programming

DP formulation:
State: dp[i] = length of LIS ending at index i
Transition: dp[i] = max(dp[j]+1) ∀ j<i, A[j]<A[i]
Base case: dp[i] = 1
Answer: max(dp[0..n-1])
Optimal Substructure: The LIS ending at i can be extended from the LIS ending at any j where A[j] < A[i]. The optimal choice at i depends on the optimal choices for all j < i.
Overlapping Subproblems: dp[j] is used by every future dp[i] (where A[j] < A[i]). Without memoization, recomputing these overlapping subproblems leads to exponential time — storing dp[] makes it O(n²).
LIS connects to: Longest Common Subsequence (LCS), Edit Distance, Box Stacking, Chain of Pairs, and Patience Sorting.
